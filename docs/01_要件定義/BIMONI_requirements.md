## 1. サービス概要

BIMONIは既に運営中の美容モニターサービスです。モニター（一般ユーザー）が美容サロン・商品の体験案件に応募し、体験・報告を行うことでモニター協力金（ポイント）を得られる仕組みです。

### 1.1 現行運用（参考情報・移行元）

| 機能 | 現行ツール |
|------|----------|
| 集客・ユーザー管理 | LINE公式アカウント + エルメ（初回アンケートフォーム） |
| ユーザーID管理 | エルメ回答者ID |
| 案件掲載 | Xサーバー上の案件一覧ページ |
| 応募受付 | エルメの応募フォーム（案件ごとに個別フォーム） |
| 応募・ポイント管理 | Googleスプレッドシート |
| 当選連絡・日程打診 | LINE（手動） |

**現行の課題（最も工数がかかっている業務）:**
1. 応募管理（エルメ＋スプレッドシートの突き合わせ）
2. 打診・案内管理（LINE手動連絡）
3. 日程調整管理
4. モニター報告確認
5. ポイント付与管理（月末締め・翌月10日付与）

### 1.2 新システムの目的

現行のエルメ・Googleフォーム・Googleスプレッドシートに依存した運用を廃止し、BIMONI独自の会員基盤・管理システムに一本化する。

## 2. 設計方針

### 2.1 認証・ユーザー識別

- **ログイン方式:** LINE LIFFログインを唯一の認証手段とする
- **主キー:** BIMONI独自の `users.id`（BIGINT AUTO INCREMENT）
- **LINE UID:** `line_user_id` として一意に保存（NULL不可）
- **エルメ回答者ID:** 主キーとして使用しない。既存データ移行用の外部IDとして `erme_respondent_id` に保存

### 2.2 優先設計方針

- **管理者向け機能を先に設計・実装する**（モニター向けより優先）
- 現行スプレッドシート管理業務の代替を最優先目標とする
- 将来的にGoogleスプレッドシートからデータをインポートできる設計

### 2.3 移行方針

- 既存ユーザーデータ（エルメ回答者ID、名前、性別、生年月日など）をインポートできる
- 既存応募履歴・ポイント履歴もインポートできる
- 新規ユーザーはLINE LIFFログインで新規登録
- 既存ユーザーはLINEログイン後にエルメ回答者IDで紐付け、または手動統合

## 3. 案件種別

BIMONIには以下の3種別があり、種別ごとに応募後フローが異なる。

| 種別 | 概要 | 応募後フロー |
|------|------|------------|
| **① 体験モニター** | 店舗へ来店して体験・資料請求・口座開設などを行う | 来店日程調整 → 来店体験 → 報告 |
| **② 商品モニター** | 商品を購入または受け取り、自宅で利用後に報告 | 商品発送 → 受取確認 → 利用 → 報告 |
| **③ 回収サービス** | 商品受取後、指定先へ返送して報告 | 商品発送 → 受取確認 → 返送 → 報告 |

## 4. 案件の管理項目

### 4.1 案件共通項目

| 項目 | 説明 |
|------|------|
| 案件名 | 案件タイトル |
| 案件種別 | 体験 / 商品 / 回収 |
| ステータス | 下書き / 公開中 / 終了 |
| PR媒体 | 掲載・PR先 |
| 開始日・終了日 | 募集期間 |
| 案件単価 | 運営の売上単価 |
| 初回購入費 | モニターが最初に払う金額 |
| 継続購入費 | 継続時の費用 |
| モニター協力金 | モニターへの報酬金額 |
| 紹介単価 | 紹介報酬単価 |
| 継続率 | 参考データ |
| 粗利 | 運営の粗利（管理者のみ閲覧） |
| 男女比 | 応募者の目標比率 |
| 注意事項 | 解約・回収・その他 |
| 募集人数 | 定員 |
| 打診予定数 | 当選候補として打診する人数 |
| 写真 | サムネイル画像 |

## 5. 応募フロー

```
案件公開
    ↓
モニター応募（LIFF画面）
    ↓
運営が応募者一覧で選考（当選 / 落選）
    ↓
当選者へLINE案内（通知 or 手動）
    ↓
日程打診（実施可能な時間帯の確認）
    ↓
日程確定
    ↓
モニター実施体験
    ↓
モニター報告（画像 + テキスト）
    ↓
運営確認・承認 or 差戻し
    ↓
モニター協力金付与（月末締め → 翌月10日付与）
```

## 6. 機能要件

### 6.1 管理者向け機能（優先実装）

#### Phase 1: LIFFログイン・会員基盤
- LINE LIFFログイン
- LINE UID取得・保存
- BIMONI会員登録フロー
- プロフィール登録（名前、フリガナ、生年月日、性別、実施可能な時間帯、継続希望）
- エルメ回答者ID保存欄
- 既存データ移行を考慮したユーザー管理

#### Phase 2: 案件管理
- 案件登録（種別・全項目対応）
- 案件編集・削除
- 案件公開 / 非公開切り替え
- 案件一覧・検索・フィルター
- 案件種別管理
- 募集人数・募集状況管理
- 注意事項管理
- モニター協力金・粗利管理

#### Phase 3: 応募管理・打診管理
- 案件応募受付
- 応募一覧・検索
- 応募履歴管理
- 当選 / 落選管理
- 選考ステータス管理
- LINE案内状況管理
- 打診ステータス管理（打診中 / 日程確定 / キャンセル）
- 日程確定管理

#### Phase 4: モニター報告管理
- 体験報告登録（モニター側）
- 画像アップロード
- 報告内容確認（管理者側）
- 承認 / 差戻し
- 案件完了管理

#### Phase 5: 協力金・ポイント管理
- モニター協力金付与
- ポイント履歴管理
- 残高管理
- 手動調整
- 付与理由管理
- 承認者管理
- 月末締め処理
- 翌月10日付与管理

#### Phase 6: LINE通知連携
- 応募確認通知
- 当選通知
- 日程調整通知
- 報告依頼通知
- ポイント付与通知
- 一括通知送信

#### Phase 7: 既存データ移行
- Googleスプレッドシートからのユーザーインポート（CSV）
- 応募履歴インポート
- ポイント履歴インポート
- エルメ回答者IDとの紐付け
- LINE UIDとの紐付け
- 重複データ統合

### 6.2 モニター向け機能（Phase 1〜5 に分散実装）

- LINE LIFFログイン・会員登録
- プロフィール登録・編集
- 案件一覧・検索
- 案件詳細確認・応募
- 応募状況確認（マイページ）
- 体験報告投稿
- 協力金・ポイント残高確認
- 協力金交換申請

## 7. 非機能要件

| 項目 | 内容 |
|------|------|
| **認証** | LINE LIFFログインのみ（メールアドレス認証は持たない） |
| **対応デバイス** | スマートフォン優先（LIFF）。管理者画面はPC対応 |
| **セキュリティ** | CSRF対策、XSS対策、LINE Signature検証 |
| **通知** | LINE Messaging API（プッシュ通知） |
| **画像アップロード** | 報告画像・案件サムネイル（Laravel Storage） |
| **言語** | 日本語 |
| **データ移行** | CSVインポート機能を提供 |
