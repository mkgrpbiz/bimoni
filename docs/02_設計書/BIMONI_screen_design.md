## 2. 管理者画面の詳細

---

### A-01 ダッシュボード

**目的:** 運営状況を一目で把握する

**表示要素:**
- KPIカード
  - 登録モニター数（総数・今月新規）
  - 公開中案件数
  - 今月の応募数
  - 承認待ち報告数
  - 未処理の打診数（日程確定待ち）
  - 今月の協力金付与合計
- 今月の締め状況（締め済み / 未締め）
- 対応が必要な一覧（最新5件ずつ）
  - 承認待ち報告
  - 日程確定待ちの応募
  - 当選通知未送信の応募

---

### A-02 案件一覧

**目的:** 全案件の状況を管理する

**表示要素:**
- フィルター: ステータス / 案件種別 / 期間
- 案件テーブル
  - 案件名 / 種別バッジ / ステータスバッジ
  - 募集人数 / 応募数 / 当選数
  - 募集期間
  - 操作（編集 / 公開切替 / 応募者一覧へ）

---

### A-03 / A-04 案件作成・編集

**入力項目（全種別共通）:**

| 項目 | 入力形式 |
|------|---------|
| 案件名 | テキスト（必須） |
| 案件種別 | セレクト（体験 / 商品 / 回収） |
| カテゴリ | セレクト |
| タグ | 複数選択 |
| ステータス | セレクト（下書き / 公開 / 終了） |
| PR媒体 | テキスト |
| 説明文 | テキストエリア |
| 応募条件 | テキストエリア |
| 注意事項 | テキストエリア（解約・回収・その他） |
| 商品名 | テキスト |
| 商品金額 | 数値（円） |
| モニター協力金 | 数値（円）（必須） |
| 紹介単価 | 数値（円） |
| 案件単価 | 数値（円）（管理者のみ） |
| 初回購入費 | 数値（円） |
| 継続購入費 | 数値（円） |
| 粗利 | 数値（円）（管理者のみ） |
| 継続率 | 数値（%） |
| 男女比 | テキスト（例: 男:女=3:7） |
| 募集人数 | 数値 |
| 打診予定数 | 数値 |
| 募集開始日・終了日 | 日付 |
| サムネイル画像 | 画像アップロード |

---

### A-05 応募一覧（全案件横断）

**目的:** 全案件の応募を一覧で管理する

**フィルター:** ステータス / 案件 / 案件種別 / 期間 / LINE案内状況

**テーブル列:**
- ユーザー名 / 案件名 / 種別 / 応募日
- ステータスバッジ
- LINE案内状況
- 打診状況
- 操作（詳細 / ステータス変更）

---

### A-06 応募者選考（案件別）

**目的:** 1つの案件への応募者を選考する

**表示要素:**
- 案件名・種別・定員・打診予定数
- 応募者テーブル
  - 氏名 / フリガナ / 性別 / 年齢 / 居住エリア
  - 実施可能な時間帯
  - 継続希望
  - 応募日時
  - 現在のステータス
  - 操作（当選 / 落選）
- 一括当選・一括落選ボタン
- 選考済み人数カウンター（例: 3/10名 当選済み）

---

### A-07 打診・日程管理

**目的:** 当選後の日程調整を管理する

**表示要素:**
- ユーザー情報（名前・LINE案内状況・実施可能な時間帯）
- 打診日時の入力フォーム（複数候補を入力）
- 確定日時の設定
- LINE通知送信ボタン（打診メッセージ送信）
- ステータス変更（打診中 → 日程確定 → キャンセル）
- メモ欄

---

### A-08 ユーザー一覧

**フィルター:** ステータス / 登録経路（新規/スプシ移行） / 性別

**テーブル列:**
- BIMONI ID / 氏名 / フリガナ / 性別 / 年齢
- 登録日 / エルメ回答者ID（移行データの場合）
- ポイント残高
- 操作（詳細）

---

### A-09 ユーザー詳細

**表示要素:**
- プロフィール情報（氏名・フリガナ・性別・生年月日・居住エリア）
- LINE UID（部分表示）
- エルメ回答者ID
- ポイント残高・履歴
- 応募履歴タブ（案件名・ステータス・日付）
- 報告履歴タブ
- アカウント操作（停止 / 有効化）

---

### A-10 報告一覧

**フィルター:** ステータス（承認待ち/承認済み/差戻し） / 案件 / 期間

**テーブル列:**
- ユーザー名 / 案件名 / 報告日時 / ステータス / 操作（詳細）

---

### A-11 報告詳細・承認

**表示要素:**
- ユーザー情報・案件情報
- 報告テキスト
- 報告画像（サムネイル一覧 → クリックで拡大）
- 承認ボタン（→ ポイント付与確認ダイアログ）
- 差戻しボタン（→ 差戻し理由入力）
- 差戻し履歴（再提出の場合）

---

### A-12 ポイント管理

**表示要素:**
- ポイント付与一覧（ユーザー / 金額 / 理由 / 案件 / 付与日 / 締め状況）
- 手動付与フォーム（ユーザー選択・金額・理由）
- 手動調整フォーム（マイナス調整も可）

---

### A-13 月末締め管理

**目的:** 月末締め → 翌月10日支払の管理

**表示要素:**
- 締め月一覧（ステータス: 未締め / 締め済み / 支払済み）
- 締め対象のポイント明細
- 締め処理ボタン（確認ダイアログ付き）
- 支払完了マークボタン

---

### A-16 データインポート

**目的:** 既存スプレッドシートデータのCSVインポート

**機能:**
- ユーザーインポート（CSVアップロード → バリデーション → 確認画面 → 実行）
- 応募履歴インポート
- ポイント履歴インポート
- インポート結果レポート（成功件数 / スキップ / エラー一覧）
- 重複ユーザーの統合ガイド画面

---

## 3. モニター向け画面の詳細

---

### M-01 LIFFログイン・会員登録

**フロー:**
```
LINEアプリ内でLIFFを開く
    ↓
LINE LIFFによる自動認証（LINE UID取得）
    ↓
既存会員 → ホーム（M-03）へ
新規会員 → M-02（プロフィール登録）へ
移行対象 → エルメIDで紐付け確認
```

---

### M-02 プロフィール登録

**入力項目:**
- 氏名（必須）
- フリガナ（必須）
- 生年月日（必須）
- 性別（必須）
- 居住エリア
- 実施可能な時間帯（複数選択）
- 継続希望（はい / いいえ）
- エルメ回答者ID（任意・既存ユーザー移行用）

---

### M-03 案件一覧（トップ）

**表示要素:**
- 公開中の案件カード一覧
- 種別フィルター（体験 / 商品 / 回収）
- 案件カード: サムネイル / 案件名 / 協力金 / 募集期限 / 残席

---

### M-04 案件詳細・応募

**表示要素:**
- 案件サムネイル
- 案件名・種別・説明・条件
- 商品名・金額（商品・回収のみ）
- 協力金
- 注意事項
- 応募ボタン（未応募）/ 「応募済み」表示（応募済み）

---

### M-05 マイページ（応募一覧）

**タブ:**
- 審査中
- 当選（日程調整中 / 日程確定済み）
- 報告待ち
- 完了済み

---

### M-06 報告投稿

**入力項目:**
- 案件名（参照表示）
- 報告テキスト
- 画像アップロード（複数枚）
- 送信ボタン

---
